语音输入：typeless vs 豆包输入法

最近我使用语音输入的频率越来越高，因为我现在很多的工作都是让 AI 来帮我完成。而我让 Claude code 或者 code X 干活，需要大量的输入去跟他沟通我的需求，确保他理解我的需求。那这个时候打字反而成了制约我效率的瓶颈，因为我脑子里的想法是井喷出来的，但是我的手跟不上我打字的速度。所以这个时候我特别欣赏语音输入这一技巧，可以让我输入的效率提升 4 到 5 倍。因此我也在不断尝试新的语音输入工具，并且我感觉后面可能会尝试自己用 AI 手撸一个。

回到正题上，最近使用了 typeless 和豆包输入法两种语音输入工具，我现在这篇帖子也是使用豆包输入法来输入的。那么我综合比较下来，它们的区别有这么几个。首先，豆包输入法是全程免费的，typeless 免费账号只能用 30 天。但是可以通过切邮箱来解决。我相信过了半年以后，会有更好的平替出现。其次，两者的识别准确率都很高，但是豆包输入法更像是把我的说的话原汁原味的表达出来，因此它会更加有人味。而 typeless 呢，它会用大模型对我说的话进行整理，比如说我跟它说了三个点，那它就会用分点的形式把我的话整理出来。因此它说的话会有点像人机味比较重。但是我觉得本质上是应用场景不同的问题。像豆包输入法我就来做一些日常的回复，非常合适。但是像 typeless 呢，就很适合用来做跟大模型交流的工作，因为它会帮我把我杂乱的思想整理成有序的文本，大模型在执行的时候也会更加高效。另外，在术语识别方面，我能明显感觉到豆包的语音输入在很多方面的数据识别非常精准。

所以我现在的 workflow 经常是在手机上随便找个对话框，把话用豆包输入法记下来，然后再通过苹果生态将其共享到 Mac 上，然后喂给大模型去回答。很方便。

我个人强烈建议豆包输入法尽快推出电脑端。现在豆包电脑端上的语音输入还是太鸡肋了，完全没有手机端上好用。

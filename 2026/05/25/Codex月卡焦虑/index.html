Codex月卡焦虑

最近在同学的推荐下，搞到了一个很不错的中转站。价格怎么说呢，两百块钱一个月，每天有三百刀的 Codex 额度。（如果有需求，可以私信我中转站链接）

三百刀一天。说实话，开发体验拉满了。

以前用 Claude 的时候我其实一直有个心结——不太敢放开用。有时候让它帮我改改论文，一天下来能干到一百多刀。但那个时候的我是这么想的：为了干活出点必要的钱，可以接受。所以就这么一直用着。

但最近 Claude 的状态……怎么说呢，感觉它用心程度下降了。干活没有那么认真了。写出来的文档我看起来有点蜻蜓点水、浅尝辄止。反正就是看得云里雾里的，不够清楚。对比之下我觉得 Codex 写得更好，所以最近一直在用 Codex。

重点来了。

在用这个中转站之前，我一直是按量计费的。什么叫按量计费呢，就是我想用多少用多少，那天不想用就不用。听起来很自由对吧？

结果就是——大部分时间我都在忙，根本没空去用。每天花不了几个钱。

然后这回充了月卡。

事情就变得微妙了起来。

这个月卡每天有三百刀的额度。而我发现，自从充了月卡之后，我开始焦虑了。不是钱不够用的焦虑，是一种很东亚的焦虑——我买了这个东西，如果今天没把它用掉，就浪费了。

浪费钱。浪费生命。对不起家里人。

各种内疚感一起来了。

而这种内疚感反向迫使我去干更多的活。因为它会让我觉得，我今天要是不把额度用完，我这一天就不够努力，我就亏了。为了减少这种内疚的成本，我会尽可能多地推进工作。

我知道。这是一种很病态的思维。典型的东亚小孩式思维——把一切资源换算成成本，把一切成本换算成愧疚，再用愧疚驱动自己。

但说真的，它在我身上就是能起作用。

它让我多写了几行代码，多改了几页文档，多推进了几个搁置了挺久的事情。我不确定这是好事还是坏事，但至少在产出上看，月卡比按量计费更适合我这种需要外部压力来驱动的人。

正好在小红书上刷到过一条帖子，大意是说——珍惜你身边那些每天会把 Codex 或者 Claude Code 的 Pro 额度用满的人，那些人一定是非常上进、非常求进步的人。

虽然有点自吹自擂，但我得承认，我觉得还是有点道理的。哈哈。

不是每个人都会把自己逼到这个程度。也不是每个人需要这样逼自己。

但对我来说，月卡焦虑，好像成了一种奇怪的燃料。

CodeX桌面宠物使用指南

从 Claude Code 泄露到 CodeX 正式化，桌面宠物这个功能就这样静悄悄地来了。

CodeX 前段时间推出了宠物功能。其实这个东西在 Claude Code 那边早就被扒出来了，但 CodeX 抢先一步把它正式化了。默认给了几只可选的小东西，也支持自己定制——你给它一段描述，它帮你生成一只专属宠物，放在桌面上陪着你写代码。

听起来像是个小玩具。但试了一下，发现不只是玩具。

我的小水滴

我当时在自己的个人文件夹下，让它根据对我的了解去生成一只宠物。结果它给我搞了一只小水滴，水滴面前还挂了个照相机。

我看到的时候愣了一下，然后笑了。

它记得我跟它说过希望被提醒多喝水，也记得我说过喜欢摄影。两个点一结合，就有了这个角色。整个过程没怎么花心思——在网上找了个 skill，丢给它让它自己去学，图片生成用中转站的 GPT Image 2 接口，几下就出来了，存在默认的宠物目录 ~/.codex/pets 下。

那一瞬间我突然觉得，这不只是个放在桌面上的装饰。它是一个 AI 对你的理解，被可视化地放在了那里。

然后就火起来了

过了一段时间，同学给我发消息，我才发现 CodeX 的宠物居然已经火成了一个社区。不是那种「有人做了几个分享」的程度，是真的形成了一个宠物 Hub——有专门的网站、分类、合集、热榜。

我进去逛了一圈，实在没忍住，笑到肚子疼。

有原神 IP 的宠物，刻晴、纳西妲、钟离，一个个整得活灵活现。有绝区零 IP 的，安比、妮可、比利，连画风都对味。最离谱的是奶龙——你能想象吗，有一天你写代码的时候，桌面角落里趴着一只奶龙看着你。

还有各种圆头耄耋的小猫。那个呆滞又无辜的眼神，天女下凡天花乱坠，我一个不怎么吸猫的人都忍不住想收藏。

感兴趣的自己去逛逛吧，上面已经有不少好东西了（还有一些敏感内容，我就不多说了）：

  • petdex.crafter.run
  • codex-pets.net

总有一款你喜欢。真的。

我依旧选择了我最爱的胡桃，一种情怀，一种感觉。

笑完之后，感到一阵悲凉

不是因为功能不好。是笑完之后，我突然意识到一件事——

我不知道发给谁。

不是没有群聊。我在各个群里像拉屎一样爆笑不停地发消息，发截图，发「你看这个奶龙哈哈哈哈」，发「这个圆头耄耋什么神仙」。大家也会回，会哈哈哈，会转发。

但我心里清楚，没有人真的 get 到我在笑什么。

要完整地理解这件事有多好笑、有多抽象、又有多奇妙，对面那个人需要什么呢——他得知道 CodeX 是什么，知道 CodeX Pets 这个功能怎么回事，知道 Claude Code 那边先泄露了但 CodeX 抢先正式化这个八卦，知道中转站是什么，知道 GPT Image 2 是什么，知道奶龙这个梗，知道原神的角色，知道绝区零的画风，知道圆头耄耋，甚至还得知道一些更敏感的话题……

然后还得跟我一样，对这件事有一种「又好笑又感动又抽象」的复杂情绪。

这种人少之又少。

这不是那种「没人理解我」的委屈。这是一种很具体的、很现实的孤独——你的快乐太复杂了，复杂到没办法用几句话传递出去。就像我在之前那篇文章里写的，AI 真的改变了我的工作方式，但这件事已经越来越难讲给别人听了。

宠物的快乐是这种感受的又一个注脚：它不仅改变了你做事的方式，它还创造了一些只属于你的、只属于一小群人的快乐。而这些人散落在互联网的各个角落，你很难把他们聚到一个聊天框里。

其实这个策略挺有意思

说回功能本身。

我觉得 CodeX 做宠物这件事，策略上挺聪明的。不是那种「增加用户粘性」的产品思路——那个太无聊了。是另一种东西。

在桌面放一个宠物，一个 AI 根据对你的理解生成的宠物，或者你从社区里挑的、跟你喜欢同一个 IP 的宠物，它在你的工作流里安静地待着。你不会因为它而「提高效率」，但它会在某个瞬间让你觉得——哦，这个 AI 工具不是冷冰冰的。

它让你在使用 AI 工作的时候，多了一点点温度。

以前我们讨论 AI 工具，讨论的是能力、速度、准确率。但现在有人开始讨论它好不好玩了。这件事本身就挺有意思的。一个能让你笑出来的 AI 工具，比一个「正确率 98%」的 AI 工具，可能更能让你持续用下去。

就像我的小水滴，它不干任何活，它就在那挂着相机看着我。但它确实让我在写代码的间隙，偶尔会想一下——

哦对，该喝水了。

这就够了。
